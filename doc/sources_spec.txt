JSON sources specification.
Strings in CAPS are all meant to replaced with the respective value.
'##' denotes a comment. These comments are simply for documentation purposes.
Comments are not expected in the actual json data.

NOTE: Dependencies are expected to be non-official modscopes/mods (we don't need to know that "blah.blah" depends on
	"brl.map", because we're only managing third-party modules).

{
	"MODSCOPE": {					## e.g. "pub"
		"name": "FORMAL NAME",		## e.g. "Public modules"
		"deps": ["MODSCOPE", "MOD", ...],	## Contents can be either a modscope (e.g. "brl") or a module (e.g. "brl.filesystem").
											## This array can be absent if there are no dependencies.
											## NOTE: This value adds dependencies for /all/ of the modules in this scope.
		
		"MOD": {						## e.g. "zlib"
			"name": "FORMAL NAME",		## e.g. "zlib wrapper"
			"deps": ["MODSCOPE", "MOD", ...],	## Contents can be either a modscope (e.g. "brl") or a module (e.g. "brl.blitz").
												## This array can be absent if there are no dependencies.
			"version": "VERSION",	## The module's current version (e.g. "1.01").
			"url": "URL"			## The get-line/url for the module's source (e.g. "http://github.com/Htbaa/process.mod/tarball/master").
		},
		## more MODs ...
	},
	## more MODSCOPEs ...
}
